## MySQL Cluster (InnoDB) Setup:

MySQL InnoDB Cluster is a built-in high availability (HA) solution in MySQL (8.0+ and 8.4) that provides:

- Automatic failover
- Data replication
- Self-healing cluster
- Single-primary or multi-primary mode
- Zero external replication tools required


### Core Components:

#### 1. MySQL Server (InnoDB + Group Replication):
This is the actual database engine running on each node. It provides:
- Transaction storage (InnoDB)
- GTID replication
- Group Replication engine


#### 2. MySQL Group Replication (GR):
This is the heart of the cluster. It ensures:
- All nodes share the same data
- Automatic failover
- Consensus-based replication (majority quorum)


#### 3. MySQL Shell + AdminAPI:
This is the management layer.
- Configures replication automatically
- Checks server readiness
- Bootstraps cluster safely

### MySQL Configuration on file `/etc/my.cnf` :

**Do NOT manually configure Group Replication in `my.cnf` when using MySQL Shell InnoDB Cluster.**

_MySQL Shell will handle:_
- Group Replication setup
- Recovery configuration
- Seed management
- Local addresses


#### Common Config (All Nodes): 

```ini

[mysqld]
socket=/var/lib/mysql/mysql.sock

bind-address = 0.0.0.0

skip_name_resolve=ON
max_connect_errors=100000

# GTID
gtid_mode=ON
enforce_gtid_consistency=ON

# Binary log
log_bin=binlog
binlog_format=ROW
binlog_checksum=NONE

# Replication


# Disable engines
disabled_storage_engines="MyISAM,BLACKHOLE,FEDERATED,ARCHIVE,MEMORY"

# Group Replication
loose-group_replication_group_name="aaaaaaaa-bbbb-cccc-dddd-aaaaaaaaaaaa"
loose-group_replication_start_on_boot=off
loose-group_replication_group_seeds="db01:33061,db02:33061,db03:33061"
```

### What `flush-hosts` does:

_Clears the internal host cache:_
- removes blocked IP entries
- resets connection error counters per host
- allows the host to reconnect again


