## Percona Xtrabackup:

Percona XtraBackup is an open-source hot backup utility for MySQL-based databases. It allows you to take non-blocking physical backups of:

- MySQL Community Server ((5.7, 8.0, 8.4+))
- Percona Server for MySQL
- Some versions of MariaDB (limited / older versions)

It is widely used in production environments because it can back up large databases with minimal downtime.



> [!NOTE]  
> ⚠️ Version must match your MySQL server major version.



### Key Features: 

#### 1. Hot Backups (No Downtime):

Takes backups while the database is running, with no downtime or table locks (for InnoDB/XtraDB).
- No need to stop MySQL
- Minimal impact on applications
- Suitable for 24/7 systems


#### 2. Physical Backups:

This makes restore operations much faster than logical dumps. XtraBackup copies actual database files:
- `.ibd`
- redo logs
- undo logs
- binary logs (optional)



#### 3. Incremental Backups:

Only backs up data that changed since the last full or incremental backup, saving time and storage.
- Smaller backup size
- Faster backup
- Reduced storage usage


#### 4. Compression Backups:

Supports on-the-fly compression to reduce backup size: 
- gzip / qpress / zstd compression
- AES encryption

Useful for:
- Offsite backups
- DR sites
- Cloud storage


#### 5. Encryption Backups:

Can encrypt backups for security at rest.



#### 6. Streaming Backups:

Can stream backups directly to another server or storage (e.g., via xbstream or tar).

Can stream directly to:
- SSH
- Netcat
- S3-compatible storage


#### 7. Partial Backups:

Back up specific databases or tables.


#### 8. Point-in-Time Recovery (PITR): 

Works with binary logs for granular recovery.






### Backup Types:

| Backup Type        | Description               |
| ------------------ | ------------------------- |
| Full Backup        | a backup of all the contents of the database      |
| Incremental Backup | a backup of only the files that have changed since the last full backup  |
| Partial Backup     | Selected databases/tables |
| Compressed Backup  | Reduced size              |
| Encrypted Backup   | Secure backup             |

## Installation on Rocky Linux 8:


### Install Percona Repository:

_Install the Percona yum repository:_
```
yum install -y https://repo.percona.com/yum/percona-release-latest.noarch.rpm
```


_Enable the repository (defaults to 'release' stable channel):_
```
percona-release enable pxb-84-lts
```




_Install dependencies:_ 
```
yum install -y libev libgcrypt openssl zlib libaio
```


_Install Percona XtraBackup:_
```
yum install -y percona-xtrabackup-84
```


_To decompress backups made using `LZ4` or `ZSTD` compression algorithm, install the corresponding package:_
```
yum install -y lz4 zstd
```

## Full Backup:


_Syntax for creating a full backup with common options:_
```
xtrabackup --backup --target-dir=<path> [--user=<user>] [--password=<password>]
```


_Create MySQL Backup user:_
```
CREATE USER 'backup'@'localhost' IDENTIFIED BY 'Back#2026';
GRANT RELOAD, LOCK TABLES, PROCESS, REPLICATION CLIENT, SELECT, BACKUP_ADMIN ON *.* TO 'backup'@'localhost'; 
FLUSH PRIVILEGES;

SHOW GRANTS FOR 'backup'@'localhost';
```


### Take Full backup:
```
xtrabackup --backup --target-dir=/home/full_backup/full_backup_$(date +%F) --user=backup --password=Back#2026
```

### Prepare a full backup (make it consistent):

In Percona XtraBackup, the `--prepare` step means: **Apply transaction logs to make the backup consistent and restorable.**

A raw backup taken by XtraBackup is not immediately usable because MySQL is running while files are copied. Some transactions may be incomplete in the copied data files.

**What Happens During `--prepare`:**
- Reads copied InnoDB pages
- Reads copied redo logs
- Replays committed transactions
- Rolls back incomplete transactions
- Produces a consistent backup
- After this step, the backup becomes safe to restore.



_Syntax for preparing a full backup:_
```
xtrabackup --prepare --target-dir=<path> [--apply-log-only] [--parallel=<N>]
```


_Preparing a full backup, then restore:_
```
xtrabackup --prepare --target-dir=/home/full_backup/full_backup_2026-05-09
```





### Restore a Full Backup (copy back to datadir):

_Syntax for restoring a prepared backup:_
```
xtrabackup --copy-back --target-dir=<path> [--datadir=<path>]
```


_Syntax (copy back to a specific datadir):_
```
xtrabackup --copy-back --target-dir=/data/backups/ --datadir=/var/lib/mysql
```



_Stop MySQL:_
```
systemctl stop mysqld

rm -rf /var/lib/mysql/*
```


_Copy back to default datadir:_
```
xtrabackup --copy-back --target-dir=/home/full_backup/full_backup_2026-05-09
```



```
chown -R mysql:mysql /var/lib/mysql

systemctl start mysqld
```

## Incremental Backup:


### Full Backup:

_First Take Full Backup:_
```
xtrabackup --backup --target-dir=/home/full --user=backup --password=Back#2026
```


### Incremental Backup:


```
-- Take incremental backup based on a full backup:
xtrabackup --backup --target-dir=/home/inc1 --incremental-basedir=/home/full --user=backup --password=Back#2026


-- Take incremental backup based on a inc1 backup:
xtrabackup --backup --target-dir=/home/inc2 --incremental-basedir=/home/inc1 --user=backup --password=Back#2026
```



### Prepare Incrementals: 

```
-- Prepare: apply incremental to full:
xtrabackup --prepare --apply-log-only --target-dir=/home/full


-- Apply inc1:
xtrabackup --prepare --apply-log-only --target-dir=/home/full --incremental-dir=/home/inc1



-- Apply inc2 - Final prepare (Rolls back uncommitted transactions):
xtrabackup --prepare --target-dir=/home/full --incremental-dir=/home/inc2
```





### Restore:


```
systemctl stop mysqld
rm -rf /var/lib/mysql/*
```


```
xtrabackup --copy-back --target-dir=/home/full
```



```
chown -R mysql:mysql /var/lib/mysql

systemctl start mysqld
```
