## MySQL Master-Slave Replication:

MySQL 8.4 Master–Slave (Primary–Replica) replication setup on Rocky Linux 8.


### Architecture overview: 

- Master (Primary): 192.168.10.191
- Slave (Replica): 192.168.10.192


### Host/DNS Configuration:

_Edit file `/etc/hosts` on all nodes:_
```
192.168.10.191          db01
192.168.10.192          db02
```


### Prerequisites: 
- Two/Three separate physical or virtual servers (One Master Node and One Slave or Two Salve Node)
- MySQL 8.4 is installed and running on both servers.
- Assign unique server IDs to each MySQL instance.
- Enable binary logging on both servers.
- Create a dedicated MySQL user for replication with the necessary privileges.

## Configure MASTER (Primary):

Open your MySQL configuration file (typically `my.cnf`) on the master server. Add or modify the following lines:


```conf
[mysqld]

datadir=/var/lib/mysql
socket=/var/lib/mysql/mysql.sock

server-id = 1
bind-address = 0.0.0.0
skip_name_resolve=ON

# Binary logging (required)
log_bin=mysql-bin
binlog_format=ROW
binlog_expire_logs_seconds = 604800   ## 7 days

# Recommended additions
log_replica_updates = ON   ## Required if this master was itself a replica, or for chain replication

# GTID-based (recommended)
gtid_mode=ON
enforce_gtid_consistency=ON

# Performance
sync_binlog = 1
innodb_flush_log_at_trx_commit = 1

log-error=/var/log/mysqld.log
pid-file=/var/run/mysqld/mysqld.pid

```



```
systemctl restart mysqld
```


```
netstat -tlpn | grep 3306

tcp        0      0 0.0.0.0:3306     0.0.0.0:*     LISTEN      8491/mysqld
```



### Create replication user:

## Configure SLAVE (Replica):


### On Slave-1:

Add or modify the following lines to the configuration file:


```conf
[mysqld]

datadir=/var/lib/mysql
socket=/var/lib/mysql/mysql.sock

server-id = 2
bind-address = 0.0.0.0
skip_name_resolve=ON

# Relay log 
relay_log=relay-bin
relay_log_recovery=ON    ## Prevents relay log corruption after crash

# Binlog
log_bin=mysql-bin
log_replica_updates = ON  ## Essential if this slave could ever become a master (failover)

# GTID
gtid_mode=ON
enforce_gtid_consistency=ON

# Make slave read-only (best practice)
read_only=ON
super_read_only=ON

log-error=/var/log/mysqld.log
pid-file=/var/run/mysqld/mysqld.pid

```

systemctl restart mysqld

## Setup Replication on slave (GTID-based):

If using GTID (recommended), the position is auto-tracked — you may not need File/Position manually.


### On Slave-1:

Log in to MySQL on the slave node and replace with  the appropriate log file and position:
- `SOURCE_HOST` 
- `SOURCE_USER`
- `SOURCE_PASSWORD` 
- `SOURCE_LOG_FILE`
- `SOURCE_LOG_POS` 
- `GET_SOURCE_PUBLIC_KEY` or `SOURCE_SSL=1`



```
STOP REPLICA;
```


```
CHANGE REPLICATION SOURCE TO
SOURCE_HOST='192.168.10.191',
SOURCE_USER='repl',
SOURCE_PASSWORD='Repl#2026',
SOURCE_LOG_FILE='mysql-bin.000001',
SOURCE_LOG_POS=862,
GET_SOURCE_PUBLIC_KEY=1;
```


Or,


_Pure GTID Mode (Recommended):_
```
CHANGE REPLICATION SOURCE TO
SOURCE_HOST='192.168.10.191',
SOURCE_USER='repl',
SOURCE_PASSWORD='Repl#2026',
SOURCE_AUTO_POSITION=1,
GET_SOURCE_PUBLIC_KEY=1;
```




_Start replication:_
```
START REPLICA;
```

## Troubleshoot:


### On Master:

_What it does (equivalent to old `RESET MASTER`) :_
- Deletes all binary logs
- Resets binlog index to mysql-bin.000001
- Clears GTID execution history
- Requires full replica rebuild


```
RESET BINARY LOGS AND GTIDS;
```


```
SHOW BINARY LOGS;

SHOW BINARY LOG STATUS;
```



### On Slave:

_What it does:_
- Stops replication
- Deletes relay logs
- Removes source connection config


_Instead of touching primary, fix the replica:_
```
STOP REPLICA;
RESET REPLICA ALL;
```




### Key MySQL 8.4 Changes:

| Old Syntax (≤8.0) | New Syntax (8.4+) |
| ----------------- | ----------------- |
| SHOW SLAVE STATUS | SHOW REPLICA STATUS | 
| CHANGE MASTER TO | CHANGE REPLICATION SOURCE TO | 
| START SLAVE | START REPLICA | 
| STOP SLAVE | STOP REPLICA | 
| SOURCE_LOG_POS | same | 
| MASTER_AUTO_POSITION | SOURCE_AUTO_POSITION | 





This sets up a fully working GTID-based MySQL 8.4 Master-Slave replication on Rocky Linux 8. GTID mode is strongly recommended as it makes failover and slave reconfiguration much simpler than file/position-based replication.
