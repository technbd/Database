## MySQL Point-in-Time Recovery (PITR):

Point-in-time recovery (PITR) is a database restoration technique that allows you to recover your database to a specific moment in time, typically up to the moment just before an error occurred or data was lost.

Before you can perform point-in-time recovery, you need to ensure that binary logging is enabled in MySQL. Binary logging records changes to the database in a binary log file, which is essential for PITR.


### Verify Binary Logging is ON: 

```
SHOW VARIABLES LIKE 'log_bin';

+---------------+-------+
| Variable_name | Value |
+---------------+-------+
| log_bin       | ON    |
+---------------+-------+
```


```

### Apply Binary Logs Until Just Before Disaster: 

| Option | Purpose |
| ------ | ------- |
| `--start-datetime` | Begin replaying logs from this time |
| `--stop-datetime` | Stop replaying before this time |
| `--start-position` | Begin at exact binlog byte position |
| `--stop-position` | Stop at exact binlog byte position | 
| `--single-transaction` | Consistent backup without locking (InnoDB) |
| `--flush-logs` | Start fresh binlog after backup | 


_Using timestamp (recover up to position before DELETE):_
```
mysqlbinlog --start-datetime="2026-05-09 18:02:41" --stop-datetime="2026-05-09 18:02:47" --skip-gtids -vv /var/lib/mysql/mysql-bin.000018 | mysql -u root -p
```


_Or, use position-based (more precise):_
```
mysqlbinlog --start-position=2112 --stop-position=2313 --skip-gtids -vv /var/lib/mysql/mysql-bin.000018 | mysql -u root -p
```



```
mysqlbinlog --start-datetime="2026-05-09 18:02:41" --stop-datetime="2026-05-09 18:02:47" --skip-gtids -vv /var/lib/mysql/mysql-bin.000018 > /home/restore1.sql

mysql -u root -p db2 < /home/restore1.sql
```

## MySQL Point-in-Time Recovery (PITR):



### PITR Flow Summary:

```
Full Backup
     +
Binary Logs
     +
Replay Until Specific Time/Position
     =
Point-in-Time Recovery
```

## Binlog2SQL:

- Python 2.7, 3.4+
- MySQL 5.6, 5.7, 8+ 

### MySQL server: 

```
[mysqld]
server_id = 1

log_bin = mysql-bin
binlog_format = row

max_binlog_size = 1G
binlog_row_image = full
```


### Install Binlog2SQL:

```
git clone https://github.com/danfengcao/binlog2sql.git
```


```
cd binlog2sql
pip install -r requirements.txt
