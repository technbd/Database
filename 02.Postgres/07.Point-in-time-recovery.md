## PostgreSQL Point-in-Time Recovery (PITR):

PostgreSQL Point-in-Time Recovery (PITR) allows you to restore a database cluster to a specific timestamp, transaction, or recovery point using:

- Base Backup
- WAL (Write Ahead Log) Archiving

_This is commonly used for:_
- Accidental DELETE/DROP
- Data corruption
- Rollback to a known good state
- Disaster recovery



### Enable WAL Archiving:

_Edit file `postgresql.conf`:_
```conf 
wal_level = replica
archive_mode = on
#archive_command = 'cp %p /tmp/pgsql/18.1/archive/%f'
archive_command = 'test ! -f /tmp/pgsql/18.1/archive/%f && cp %p /tmp/pgsql/18.1/archive/%f'   ## Prevents overwriting archived WAL
archive_timeout = 60   ## Forces WAL switch every 60 seconds
max_wal_senders = 5
wal_keep_size = 128MB
```


```
mkdir -p /tmp/pgsql/18.1/archive
chown -R postgres:postgres /tmp/pgsql/18.1/archive
chmod -R 700 /tmp/pgsql/18.1/archive
```


```
SHOW archive_mode;

SHOW archive_command;
```

### Take Base Backup:

```
mkdir -p /tmp/pgsql/base_backup1
```


```
pg_basebackup -h localhost -p 5432 -U postgres -D /tmp/pgsql/base_backup1 -P

Or,

pg_basebackup -h localhost -p 5432 -U postgres -D /tmp/pgsql/base_backup2_$(date +%Y%m%d) -Fp -X stream -P
```

### Find Recovery Time:

_Check PostgreSQL logs:_
```
grep -i delete /u01/pgsql/diag/trace/postgresql-2026-05*

2026-05-10 11:46:15.634 +06 [822242] LOG:  statement: delete from t1 where id > 2;
```


Recovery target (Set recovery target to 1 second before the delete):
```
2026-05-10 11:46:14
```



### Stop PostgreSQL:

```
pg_ctl stop -D $PGDATA
```


### Restore Base Backup:

_Move current data directory:_
```
mv /u01/pgsql/18.1/dbs/data /u01/pgsql/18.1/dbs/data_old
```


_Restore backup:_
```
cp -r /tmp/pgsql/base_backup1 /u01/pgsql/18.1/dbs/data


Or, (Better rsync for Preserves permissions)

rsync -a /tmp/pgsql/base_backup1/ /u01/pgsql/18.1/dbs/data
```


```
chown -R postgres:postgres /u01/pgsql/18.1/dbs/data
chmod -R 0750 /u01/pgsql/18.1/dbs/data
```


### Configure Recovery:

_Edit file `/u01/pgsql/18.1/dbs/data/postgresql.auto.conf` :_

```conf
restore_command = 'cp /tmp/pgsql/18.1/archive/%f %p'
recovery_target_time = '2026-05-10 11:46:14'

#recovery_target_inclusive = false
recovery_target_action = 'promote'  ## Avoids manual pg_ctl promote
```


_Create recovery signal file:_
```
touch /u01/pgsql/18.1/dbs/data/recovery.signal
```


### Start PostgreSQL:

```
pg_ctl start -D $PGDATA
```


### Verify Recovery:

_All 4 rows restored — recovery successful:_
```
select * from t1;

 id | name
----+------
  1 | aaa
  2 | bbb
  3 | ccc
  4 | ddd
```



### Check Recovery Status:

_Until promoted:_
- Database is read-only
- WAL functions unavailable
- No writes allowed


```
SELECT pg_is_in_recovery();

 pg_is_in_recovery
-------------------
 t
(1 row)
```


_Method-1 (Recommended):_
```
pg_ctl promote -D $PGDATA
```


_Method-2 (Inside SQL):_
```
SELECT pg_promote();
```


```
SELECT last_value FROM t1_id_seq;

```


_Optional, Reset sequence manually (Gaps are normal, expected, harmless):_
```
SELECT setval('t1_id_seq', 4);
```


_Better safer version:_
```
SELECT setval(
    't1_id_seq',
    (SELECT MAX(id) FROM t1)
);
```


_Insert Data:_
```
insert into t1 (name) values ('eee');
```



### Useful WAL Commands:

_Current WAL File:_
```
SELECT pg_current_wal_lsn();
```



_Force WAL Switch:_
```
SELECT pg_switch_wal();
```


_Check Archiver Status:_
```
SELECT * FROM pg_stat_archiver;

SELECT archived_count, last_archived_wal, last_archived_time, failed_count FROM pg_stat_archiver;
```

## Creates a named restore point:


### Step-1: Create Restore Point:

When you run:
- writes a special record into WAL
- saves the label `before_delete`
- remembers the exact recovery position (LSN)

```
SELECT pg_create_restore_point('before_delete');

 pg_create_restore_point
-------------------------
 0/E0001D0
(1 row)
```


### Step-2: Accident Happens:

```
DELETE FROM t1;
```


### Step-3: Restore Base Backup:

_Stop Database:_
```
pg_ctl stop -D $PGDATA
```


_Move current data directory:_
```
mv /u01/pgsql/18.1/dbs/data /u01/pgsql/18.1/dbs/data_old
```


_Restore base backup:_
```
rsync -a /tmp/pgsql/base_backup2_20260510/ /u01/pgsql/18.1/dbs/data
```


```
chown -R postgres:postgres /u01/pgsql/18.1/dbs/data
chmod -R 0750 /u01/pgsql/18.1/dbs/data
```




### Step-4: Configure PITR Recovery:

_Edit file `/u01/pgsql/18.1/dbs/data/postgresql.auto.conf` :_
```conf
restore_command = 'cp /tmp/pgsql/18.1/archive/%f %p'
#recovery_target_time = '2026-05-10 11:46:14'
recovery_target_name = 'before_delete'

#recovery_target_inclusive = false
recovery_target_action = 'promote'  ## Avoids manual pg_ctl promote
```


_Create recovery signal file:_
```
touch /u01/pgsql/18.1/dbs/data/recovery.signal
```


```
pg_ctl start -D $PGDATA
```


### Step-5: Verify:

```
SELECT pg_is_in_recovery();

 f
(1 row)
```
