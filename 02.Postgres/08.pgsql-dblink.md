
## PostgreSQL DBlink:


### Architecture Overview:

```
Local Server (192.168.10.100)          Remote Server (192.168.10.193)
──────────────────────────────         ──────────────────────────────
Database : rc_int                      Database : fielddb

User     : postgres, gob               User     : fuser

Table    : t1 (local data)             Table    : t1 (remote data)

Foreign Table  : ft1 ──── postgres_fdw ────► t1
```



### DBlink vs postgres_fdw: When to Use Which:

| Feature                    | `dblink`                        | `postgres_fdw` + Foreign Table     |
|----------------------------|---------------------------------|------------------------------------|
| Setup complexity           | Minimal                         | Server + mapping + foreign table   |
| Query style                | Verbose function call           | Plain SQL (`SELECT * FROM ft1`)    |
| JOIN with local tables     | Awkward                         | Natural — just JOIN normally       |
| Performance                | Basic                           | Better (pushes WHERE to remote)    |
| Write support              | Yes (`dblink_exec`)             | Yes (INSERT/UPDATE/DELETE on ft1)  |
| Best for                   | One-off / dynamic / ad-hoc      | Regular, repeated access           |
| Column type declaration    | Required every query            | Declared once at table creation    |

## Install PostgreSQL:

_Install PostgreSQL Repo:_
```
dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-8-x86_64/pgdg-redhat-repo-latest.noarch.rpm
```


_Disable the built-in PostgreSQL module:_
```
dnf -qy module disable postgresql
```


_Install PostgreSQL (`postgresql16-contrib` is required):_
```
dnf install -y postgresql16-server postgresql16-contrib 
```


_Initialize the database:_
```
/usr/pgsql-16/bin/postgresql-16-setup initdb

Initializing database ... OK
```


```
systemctl enable postgresql-16
systemctl start postgresql-16
```



_Verify Extension Files Exist:_
```
find / -name dblink.control
/usr/pgsql-16/share/extension/dblink.control
```


```
find / -name postgres_fdw.control 
/usr/pgsql-16/share/extension/postgres_fdw.control
```



_Edit file `postgresql.conf`:_
```conf
listen_addresses = '*'
```


_Edit file `pg_hba.conf`:_
```conf
local   all             all                                     trust

host    all             all             192.168.10.0/24         trust   #scram-sha-256
```



```
systemctl restart postgresql-16
```

## Create Database: 


### Remote Database Setup:


_Login to remote PostgreSQL server:_
```
psql -U postgres
```


_Create remote user and database:_
```
create user fuser password 'fuser';

create database fielddb owner fuser;
```


_Connect as `fuser` to `fielddb`:_
```
psql -U fuser -d fielddb


CREATE TABLE t1 (id integer PRIMARY KEY, name text);

INSERT INTO t1 VALUES (1, 'Remote-Apple');
INSERT INTO t1 VALUES (2, 'Remote-Mango');

select * from t1;
```



### Local Database Setup: 

_Login to local PostgreSQL server:_
```
psql -U postgres
```


_Create local database and additional user:_
```
create database rc_int;

create user gob password 'gob';
```


_Connect to local database:_
```
psql -U postgres -d rc_int


CREATE TABLE t1 (id integer PRIMARY KEY, name text);

insert into t1 values (1, 'Apple');
insert into t1 values (2, 'Mango');

select * from t1;
```





> [!NOTE]  
> Goal: Access data in remote `fielddb` from local `rc_int`. Create DBlink on Local Database (`rc_int`).

```
psql -U postgres -d rc_int
```

### Create Foreign Server:

```
psql -U postgres -d rc_int


CREATE SERVER field_srv_remote 
FOREIGN DATA WRAPPER postgres_fdw
OPTIONS (host '192.168.10.193', dbname 'fielddb', port '5432');
```


_If Delete:_
```
DROP SERVER field_srv_remote CASCADE;
```


_Verify:_
```
SELECT * FROM pg_foreign_server;

  oid  |     srvname      | srvowner | srvfdw | srvtype | srvversion | srvacl |                   srvoptions
-------+------------------+----------+--------+---------+------------+--------+------------------------------------------------
 16606 | field_srv_remote |       10 |  16602 |         |            |        | {host=192.168.10.193,dbname=fielddb,port=5432}
(1 row)
```


_Grant usage to local users who need remote access:_
```
GRANT USAGE ON FOREIGN SERVER field_srv_remote TO postgres;

//GRANT USAGE ON FOREIGN SERVER field_srv_remote TO gob;
```

### Create User Mapping:

```
psql -U postgres -d rc_int


-- Syntax:
CREATE USER MAPPING FOR CURRENT_USER
SERVER field_srv_remote
OPTIONS (user 'remote_user', password 'remote_password');


-- Create Map local postgres → remote fuser:
CREATE USER MAPPING FOR postgres
SERVER field_srv_remote
OPTIONS (user 'fuser', password 'fuser');


-- Map local gob → remote fuser (if gob needs access)
CREATE USER MAPPING FOR gob
SERVER field_srv_remote
OPTIONS (user 'fuser', password 'fuser');
```


_If Delete:_
```
DROP USER MAPPING FOR postgres SERVER field_srv_remote;
```


_Verify:_
```
select * from pg_user_mappings;

-------+-------+------------------+--------+----------+-----------------------------

(1 row)
```




### Create Named Connection (via Foreign Server): 

_Create Named dblink Connection via foreign server:_
```
select dblink_connect ('my_dblink', 'field_srv_remote');


 dblink_connect
----------------
 OK
(1 row)
```


```
SELECT * FROM dblink_get_connections();


 dblink_get_connections
------------------------
 {my_dblink}
(1 row)
```




#### Query via Named Connection:

_Read from the Remote Table:_
```
SELECT * FROM dblink('my_dblink', 'select * from t1') AS t(id integer, name text);

 id | name
----+------
  3 | aaa
(1 row)
```


#### Insert via Named Connection: 

_Insert into Remote Table: (Inner single quotes must be escaped as ''aaa'') :_
```
SELECT dblink_exec ('my_dblink', 'insert into t1 (id, name) values (3, ''aaa'')' );
```




_Close Named Connection:_
```
SELECT * FROM dblink_disconnect('my_dblink');

SELECT dblink_disconnect('my_dblink');
```

_Read using inline connection string:_
```
SELECT * FROM dblink(

'select * from t1') AS t(id integer, name text);

 id | name
----+------
  3 | aaa
(1 row)
```





### Create the foreign table (Optional): 

`ft1` is a fake local table — it has no data. It's just a pointer. When you query `ft1`, PostgreSQL secretly fetches from `t1` on the remote server.

```
CREATE FOREIGN TABLE ft1 (id integer, name text)
  SERVER field_srv_remote
  OPTIONS (schema_name 'public', table_name 't1');
```


_Query it like any local table:_
```
SELECT * FROM ft1;
```

_You can also insert, update, delete — it affects the remote table directly:_
```
INSERT INTO ft1 VALUES (4, 'Remote-Banana');   -- inserts into remote t1
```


_Remove Foreign Table:_
```
DROP FOREIGN TABLE ft1;
```
