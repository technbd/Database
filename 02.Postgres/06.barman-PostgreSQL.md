## Barman:

Barman (Backup and Recovery Manager) is an open-source administration tool for disaster recovery of PostgreSQL servers written in Python. It allows your organisation to perform remote backups of multiple servers in business critical environments to reduce risk and help DBAs during the recovery phase.

This tool developed by **2ndQuadrant** (**now part of EDB**) for managing backup and recovery of PostgreSQL databases.

It supports:
- Disaster recovery
- Point-in-time recovery (PITR)
- Remote backups
- WAL archiving
- Incremental backups
- Streaming replication integration


### Barman Components:
- **Barman Server**: A dedicated host where backups and WALs are stored.
- **PostgreSQL Servers**: **Primary** or **standby** nodes being backed up.
- **SSH + rsync**: Used for file transfer and remote commands.
- **Streaming Replication** (optional): Uses `pg_receivewal` or replication slot for WAL collection.


### Backup Types:
- **Full Base Backup**: Captures full data directory
- **Incremental Backup**: Based on changes since last backup
- **Streaming Backup**: Uses replication protocol for real-time WAL streaming


### Recovery Modes:
- **Full Recovery**: Restore the full database
- **Point-in-Time Recovery**: Restore to a specific point using base + WAL
- **Remote Recovery**: Push backup directly to another server (e.g., new DB node)



### Prerequisites:
- Linux operating system (Debian, Ubuntu, RHEL, Rocky, Fedora, etc.)
- Python 3.8 or higher
- Time sync via `chronyd` or `ntpd`
- PostgreSQL client tools: Required to interact with the Postgres server
- Both node **replication Configured** and check replication `select pg_is_in_recovery();`
- Password-less SSH from **barman** hosts to both **db01** and **db02** as `postgres` user

create role barman with login superuser password 'barman';

The main configuration file (set to `/etc/barman.conf` by default) contains general options such as main directory, system user, log file, and so on.

_On barman node `/etc/barman.conf`:_
```conf 

[barman]
barman_user = barman

configuration_files_directory = /etc/barman.d

barman_home = /var/lib/barman

log_file = /var/log/barman/barman.log

log_level = INFO

; Default compression level: possible values are None (default), bzip2, gzip, pigz, pygzip or pybzip2
compression = gzip
```





### Create a config file for `db01`:

The following Barman features depend on PostgreSQL binaries:

- Streaming backup with `backup_method = postgres` (requires `pg_basebackup`)
- Streaming WAL archiving with `streaming_archiver = on` (requires `pg_receivewal` or `pg_receivexlog`)
- Verifying backups with `barman verify-backup` (requires `pg_verifybackup`)
- All binaries are installed in `/usr/pgsql-16/bin` or `/usr/edb/as16/bin`



_Create file on postgres-based `/etc/barman.d/db01.conf`:_

```conf 

[db01]
description = "Primary PostgreSQL Server"
conninfo = host=db01 port=5444 user=barman dbname=postgres password=barman
streaming_conninfo = host=db01 port=5444  user=streaming_barman password=barman

backup_method = postgres
streaming_archiver = on
slot_name = barman
create_slot = auto

streaming_backup_name = barman_streaming_backup
streaming_archiver_name = barman_receive_wal
streaming_archiver_batch_size = 50

#retention_policy = RECOVERY WINDOW OF 7 DAYS

active = true
path_prefix = /usr/edb/as16/bin
```



_For rsync-based backup:_
```conf 

[db01]
description = "Primary PostgreSQL Server"

conninfo = host=db01 port=5444 user=barman dbname=postgres password=barman

backup_method = rsync
backup_options = concurrent_backup
archiver = on

#ssh_command = ssh postgres@<db-server-ip>
ssh_command = ssh enterprisedb@192.168.10.191

path_prefix = /usr/edb/as16/bin
active = true

```



_As user barman:_
```
echo 'export PATH=/usr/edb/as16/bin:$PATH' >> ~/.bash_profile
```

## Backup:


### Streaming backup:

_To start a backup you can use the `barman backup` command::_
```
barman backup <server_name>

barman backup db01

Or,

barman backup db01 --wait
```


```
barman list-backups db01


db01 20260202T215029 - F - Mon Feb  2 15:49:54 2026 - Size: 272.8 MiB - WAL Size: 0 B - WAITING_FOR_WALS
```



```
barman show-backup db01 <backup_id>

barman show-backup db01 20260202T215029
```


```
barman check-backup <server_name> <backup_id>

barman check-backup db01 20260202T215029
```



### Delete Backup: 

_Verify barman process:_
```
barman list-processes db01


Active subprocesses for server db01:
45584 receive-wal
```


_Stop Barman WAL streaming:_
```
barman receive-wal --stop db01
```



_Delete a backup:_
```
barman delete <server_name> <backup_id>

barman delete db01 20260202T215029
```



### Schedule automatic full backups:

## Restore:

### Restore to specific location (Dry-run):

```
mkdir -p /tmp/db01_restore_test
```


```
barman recover <server_name> <backup_id> /path/to/recover/dir

barman recover db01 20260202T215029 /tmp/db01_restore_test
```



### Restore to PostgreSQL Server:


_On a new server or wiped data directory: (on primary)_
```
systemctl stop edb-as-16

rm -rf /var/lib/edb/as16/data/*
```


_On Barman Server:_
```
barman list-backup db01

db01 20260204T122558 - R - Wed Feb  4 12:26:01 2026 - Size: 275.5 MiB - WAL Size: 16.1 KiB
db01 20260204T121014 - R - Wed Feb  4 12:10:15 2026 - Size: 275.5 MiB - WAL Size: 32.6 KiB
```


_Run restore from Barman server to PostgreSQL:_
```
barman recover db01 20260204T121014 /var/lib/edb/as16/data --remote-ssh-command "ssh postgres@db01"

barman recover db01 20260204T121014 /var/lib/edb/as16/data --remote-ssh-command "ssh enterprisedb@db01"
```



_Delete or comment the line after restore:_
```
vim postgresql.conf

#archive_command = false
```



_On Database Server:_
```
systemctl start edb-as-16
```





### PITR (Optional):


_On Database Server:_
```
systemctl stop edb-as-16
```


_On Database Server:_
```
mv /var/lib/edb/as16/data /var/lib/edb/as16/data_broken_$(date +%Y%m%d)

mkdir -p /var/lib/edb/as16/data
```



#### Simple mental model: 

```
[ FULL BACKUP ] ---- WAL ---- WAL ---- ❌ DELETE ---- WAL
                    ↑
             recover to HERE

```


_On Barman Server:_
```
barman list-backup db01


db01 20260204T122558 - R - Wed Feb  4 12:26:01 2026 - Size: 275.5 MiB - WAL Size: 16.1 KiB
db01 20260204T121014 - R - Wed Feb  4 12:10:15 2026 - Size: 275.5 MiB - WAL Size: 32.6 KiB
```


_On Barman Server:_
```
barman check-wal-archive db01
```




_If you want PITR (point-in-time restore) on Barman server:_

- Backup `20260204T121014` ended at: `2026-02-04 12:10:15.598939`
- So your target time must be slightly after that, e.g.: `2026-02-04 12:10:16` or `2026-02-04 12:10:17`

```
barman recover db01 20260204T121014 /var/lib/edb/as16/data --target-time "2026-02-04 12:10:16" --remote-ssh-command "ssh postgres@db01"

barman recover db01 20260204T121014 /var/lib/edb/as16/data --target-time "2026-02-04 12:10:16" --remote-ssh-command "ssh enterprisedb@db01"
```



_On Database Server:_
```
rm -rf recovery.signal
```


_On Database Server:_
```
systemctl start edb-as-16
```

### When to choose Barman:

_Choose Barman if:_
- You want a central backup server
- You manage many Postgres servers
- Compliance, audit, visibility matter
- Backups are mostly full + WAL
- You prefer simple, predictable behavior
