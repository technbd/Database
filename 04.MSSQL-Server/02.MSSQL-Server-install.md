## MSSQL Server 2019 Install: 





### Requirements:

- **Operating system**:	Windows 10 TH1 1507 or greater and Windows Server 2016 or greater
- **.NET Framework**: Minimum operating system includes minimum .NET framework.
- **Processor Type**: x64 Processor: All Intel and AMD x86-64 CPUs with 64 cores or less per NUMA node
- **Processor Speed**: Minimum: x64 Processor: 1.4 GHz, Recommended: 2.0 GHz or faster
- **Hard Disk**:	SQL Server requires a minimum of 6 GB
- **Memory**: Express Editions: 1 GB, All other editions: At least 4 GB

## Verify Installation:

After installing Microsoft SQL Server, you can verify that it’s running and accessible in a few ways.

- Default instance → uses TCP 1433
- Named instance → uses dynamic ports by default

| Component                                     | Default Port        |
| --------------------------------------------- | ------------------- |
| SQL Server Database Engine (Default Instance) | **1433**            |
| SQL Server Browser                            | 1434 (UDP)          |
| Dedicated Admin Connection (DAC)              | 1434 (TCP, dynamic) |



_Check Listening Ports:_
```bash

netstat -ano 
netstat -ano | findstr 1433
netstat -ano | findstr 1434
```



### Verify Using `sqlcmd`: 

_For Default instance:_
```
sqlcmd -S localhost -E
```


_If Named Instance (example: `MSSQLSERVER2019`):_
```
sqlcmd -S DESKTOP-T54P1V9\MSSQLSERVER2019 -E
sqlcmd -S localhost\MSSQLSERVER2019 -E
sqlcmd -S .\MSSQLSERVER2019 -E
```

## Start/Stop SQL Server:

SQL Server runs as a Windows service. Check its status:

_Stop and Start **SQL Server** using `services.msc`, Press: `Win + R` :_
```
services.msc
```


_Find your instance: **SQL Server (MSSQLSERVER2019)**_

![alt text](./assets/17.png)





#### Start/Stop Alternative (Command Prompt):

```
net stop MSSQL$MSSQLSERVER2019

net start MSSQL$MSSQLSERVER2019
```

## SQL Server Management Studio (SSMS):

SQL Server Management Studio (SSMS) is the official GUI tool from Microsoft for managing, developing, and performing DBA tasks on Microsoft SQL Server.



![alt text](./assets/18.png)



### What You Can Do With SSMS:

#### Development: 
- Write & execute T-SQL queries
- Create databases & tables
- Stored procedures, views, functions
- Query execution plan analysis

#### Administration (DBA): 
- Backup & Restore
- Security & Login management
- Jobs (SQL Server Agent)
- Replication setup
- Monitoring activity
- Performance tuning

#### Analysis & Reporting:
- Connect to Analysis Services
- Manage SSRS reports

##  Create Database: 

```sql
-- Create database
CREATE DATABASE db1;


-- Switch to database
USE db1;


-- Create table
CREATE TABLE t1 (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);


-- Insert data
INSERT INTO t1 (id, name) VALUES (1, 'Jaber');
INSERT INTO t1 (id, name) VALUES (2, 'Jon');


-- Query data
SELECT * FROM t1;

```


_Describe Database db1:_
```sql
EXEC sp_helpdb 'db1';
```


_Catalog view:_
```sql
SELECT name, database_id, create_date
FROM sys.databases
WHERE name = 'db1';
```


_Describe Table t1:_
```sql
EXEC sp_help 't1';
```

Or, 

```sql
SELECT 
    COLUMN_NAME,
    DATA_TYPE,
    CHARACTER_MAXIMUM_LENGTH,
    IS_NULLABLE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_NAME = 't1';
```
